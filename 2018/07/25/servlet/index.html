#servlet
servlet在java中的基本作用示意图

servlet 是浏览器和数据库的中间一层

Java servlet是运行在带有java servlet规范的解释器的web服务器上的java类
（比如tomcat就是其中一款开源免费的来自Apache的服务器）

#servlet的生命周期

通过init函数进行初始化，service用来进行处理客户端的请求，
destroy结束执行。通过jvm垃圾回收机制回收垃圾

servlet在进行初始化时加载和创建一部分的数据，这些数据将会伴随servlet的整个生命周期

service方法，该方法是执行实际任务的主要方法。servlet容器（web服务器）
调用service方法来处理来自客户端的请求，并把格式化的响应返回给客户端
接收到一个servlet请求时，服务器会产生一个新的线程调用服务

service方法由容器调用，不用对service做改动，只需要根据客户端的请求类型重写对应的doGet或doPost方法

destroy() 方法只会被调用一次，在 Servlet 生命周期结束时被调用。destroy() 方法可以让您的 Servlet 关闭数据库连接、停止后台线程、把 Cookie 列表或点击计数器写入到磁盘，并执行其他类似的清理活动。

在调用 destroy() 方法之后，servlet 对象被标记为垃圾回收。

##servlet表单数据
很多情况下传递信息，从浏览器到web服务器，再到后台程序，从浏览器到web服务器有两种方法，get和post
get方法是默认从浏览器向web服务器传递信息的方法，会产生一个很长的字符串
会出现在浏览器的地址栏中，敏感信息不要使用get方法
get请求的字符串最多只能用1024个字符，servlet使用doGet处理这种类型的请求

###使用servlet读取表单数据

getParameter()：您可以调用 request.getParameter() 方法来获取表单参数的值。
getParameterValues()：如果参数出现一次以上，则调用该方法，并返回多个值，例如复选框。
getParameterNames()：如果您想要得到当前请求中的所有参数的完整列表，则调用该方法。
